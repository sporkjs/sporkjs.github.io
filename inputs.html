HTML Inputs

Button

The tag <button>...</button> provides a button (gasp!). Buttons are usually used to activate a JavaScript function using the onclick property. 
<button onclick="alert('ouch');">Click Me</button>
Any HTML can be placed inside a button. Here is a button with a table inside it. 
<button onclick="alert('ouch');">
<table border="1">
<tr><td>a</td><td>b</td></tr>
<tr><td>c</td><td>d</td></tr>
</table>
</button>
Buttons can be styled with css. Here is a green button with round corners. Click it and it will turn blue. 
<button id="greenButton" 
     onclick="greenButton.style.background='blue';" 
     style="border-radius:10px;background:green;">
     Click Me
</button>
Note: The onclick property can be set for most HTML elements. This way arbitrary elements (divs, tables, spans, etc.) can be made to act like buttons.

Textarea

The tag <textarea>...</textarea> is used to provide an area in which a user can type multi-line text. The size of the textarea can be set with thecols and rows properties or with css. 
<textarea cols="30" rows="5">This is a multi-line text area.</textarea>
The contents of a textarea can be accessed in JavaScript via the value attribute. The textarea in the following example has ID typedText. The contents of the text area are accessed with typedText.value. 
<textarea id="typedText" cols="30" rows="5">Type here and push the button.</textarea>
<button onclick="alert(typedText.value);">Click Me</button>

Input Tag

HTML provides a tag <input> which allows a variety of ways to gather input. These were initially intended to be used as parts of forms, but we will use them to interact with JavaScript functions.

Text

The text type of input is used for one accepting one line of text input. The width of the text input field is specified with the attribute size. Text inputs can be styled with css, and the contents of a text input can be accessed through the value attribute just as a textarea.
Type here: 
Type here: <input type="text" id="someText" size="30">

Radio

The input type radio allows the user to select at most one of a set of options. If multiple radio inputs have the same name attribute, then only one of them can be checked at a time.
Red
Blue
Green 
<input type="radio" name="color"> Red <br>
<input type="radio" name="color"> Blue <br>
<input type="radio" name="color"> Green
Each radio input has an attribute checked wich is set to true when that option is selected. This attribute can be set to force a default value so that the user has to enter exactly one option.
Red
Blue
Green 
<input type="radio" name="color" checked="true"> Red <br>
<input type="radio" name="color"> Blue <br>
<input type="radio" name="color"> Green
To select an option from the list above, you have to click the radio circle. If we give each radio input an ID, then the words adjacent to the radio input can be turned into labels which can also be clicked using the <label> tag. Try clicking on the words in the options below. 



<input type="radio" name="color" id="redRadio" checked="true"> 
<label for="redRadio">Red</label> <br>
<input type="radio" name="color" id="blueRadio">
<label for="blueRadio">Blue</label> <br>
<input type="radio" name="color" id="greenRadio">
<label for="greenRadio">Green</label> <br>
Figuring out what a user has selected using radio inputs can require some work. You may have to look at every radio input and see which one is checked. An alternative is to use onclick to set the value of a variable which will always hold the option selected. 
<input type="radio" name="color" onclick="colorSelection='red';" checked="true"> Red <br>
<input type="radio" name="color" onclick="colorSelection='blue';"> Blue <br>
<input type="radio" name="color" onclick="colorSelection='green';"> Green

Hidden

Sometimes it is useful to be able to store data in an input field which a user cannot see. For example, if you are using the onclick approach to store a radio selection, you might want to store it in a hidden input so that the information can be submitted with a form. This is demonstrated below.
Red
Blue
Green 
<input type="radio" name="color" onclick="hiddenInput.value='red';" checked="true"> Red <br>
<input type="radio" name="color" onclick="hiddenInput.value='blue';"> Blue <br>
<input type="radio" name="color" onclick="hiddenInput.value='green';"> Green
<input type="hidden"  id="hiddenInput" value='red'>

Fieldset

Several HTML inputs can be grouped together and labeled with the <fieldset>...</fieldset> and <legend>...</legend> tags.
Color Red
Blue
Green

     <legend>Color</legend>
     <input type="radio" name="color"> Red <br>
     <input type="radio" name="color"> Blue <br>
     <input type="radio" name="color"> Green
</fieldset>

Forms

If several inputs are intendend to be used to collect information and that information is all going to be processed at the same time by the same "entity" then those inputs are usually combined in a form.
Contact Information First Name:
Last Name:
email:
Gender: Female Male 
<form action='javascript:processForm();'>
	<fieldset>
		<legend>Contact Information</legend>
		First Name: <input type='text' name='firstName'><br>
		Last Name: <input type='text' name='lastName'><br>
		email: <input type='text' name='email'><br>
		Gender: <input type='hidden' id='gender'>
			<input type='radio' onclick='gender.value="female";'>Female
			<input type='radio' onclick='gender.value="male";'>Male
	</fieldset>
	<input type='submit' value='Submit'>
</form>
The action attribute of the form tells the browser what to do with the information. We are calling a JavaScript function. Frequently, the action is calling some page or program that lives on a server somewhere (which we are not doing this semester). The form ends with a submit button. The action is performed when the button is clicked. An alternative to this is to omit the action and submit button and to use an actual button with an onclick event to submit.

Name or ID?

You may have noticed that we used name= and id= in our forms. Which one should we use? If you are going to send your form information to a php page somewhere, you need to use name. Multiple inputs can have the same name. If you are not going to use php, if you only have one element with a particular name, and if you are going to reference that element with JavaScript or CSS, us ID.
